About This Project

Project Overview

This project is a book review and recommendation platform, allowing users to create accounts, review books, and receive personalized book recommendations. It combines user engagement with data-driven recommendation techniques to create a unique user experience.

Project Goals

  • Enable users to create accounts and securely log in.
  • Allow users to search for and review books.
  • Implement a recommendation system using collaborative filtering and the K-Nearest Neighbors (KNN) algorithm.

Development Steps

1. Setting Up the Development Environment

The project was developed on an Ubuntu server hosted on AWS. We set up Node.js, MySQL, and other necessary packages to manage server-side functionality and the database.

2. Designing the Database Schema

The database was created with tables for storing user information, book details, and user reviews. Key tables include:

  • userData: Stores user account details.
  • userBooks: Stores book titles, ISBNs, and metadata.
  • userRatings: Stores user reviews, ratings, and relevant book information.

3. Implementing User Authentication

User authentication was implemented using sessions to ensure secure access to the platform. This included login, registration, and session management to keep users logged in across requests.

4. Creating APIs for Book and Review Management

RESTful APIs were built using Express.js for handling book data and user reviews. Routes were created for submitting reviews, retrieving book data, and accessing user-specific content.

5. Developing the Recommendation System

The recommendation system utilizes collaborative filtering and the KNN algorithm to suggest books based on user ratings. This process involves:

  • Collecting and analyzing user rating data.
  • Calculating similarity between users to provide relevant book recommendations.

6. Integrating the Google Books API

To enrich book information, the Google Books API was integrated. When users click on book titles, detailed information is retrieved from the API and displayed, including an option to submit reviews.

7. Designing the Front-End Interface

HTML, CSS, and JavaScript were used to create a user-friendly front end. This includes the main pages such as the login page, user dashboard, and review submission page.

8. Testing and Deployment

After development, the project was thoroughly tested to ensure that all features work as expected. Finally, the project was deployed, making it accessible to users online.

Future Plans

We plan to further improve the recommendation algorithm and refine the user interface. Additional features, like social sharing and a more advanced search engine, are also under consideration.
